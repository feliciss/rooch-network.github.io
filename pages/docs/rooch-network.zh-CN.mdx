# Rooch Network

Rooch Network为Web3 DApps提供强大的一体化服务，使其能够轻松灵活地部署。这使得Web3开发者在不用顾虑基础设施问题的前提下，可以更专注于构建更个性化的产品。

## 执行层

执行层由已Rooch作为容器以及智能合约执行环境的Dapp组成。

根据不同的场景和资源需求，Rooch的使用有三种类型：


- **通用平台** - 我们将启动并维护一个Rooch实例作为通用的智能合约平台。DApps和开发人员将能够将其智能合约直接部署到现有平台上。开发体验将与在Layer1区块链上工作的体验相同，但具有更高的TPS，更低的Gas费用以及更大更方便的存储空间。
- **DApp专用平台** - 想要拥有定制平台的DApps可以启动一个专用的Rooch实例，开发者可以自定个性化的制以及特殊的特性，如通证经济模型。
- **Sector专用平台** - 想要使用多个DApps开发和扩展其生态系统的Web3开发者也可以启动Rooch实例，不同DApp的资产和功能将具有无缝互操作性，用户可以在不同生态系统DApp之间共享帐户和配置文件。

## 网络中间件

Rooch网络有几个关键的中间件服务，负责处理执行层上所有Rooch实例的交易和状态。所有服务都基于相同的核心Rooch实现。

- **排序器** - 排序器负责验证传入的交易并对其进行排序。排序后的交易将提交给DA Chain和Proposer。
- **挑战者** - 挑战者是Rooch Layer2网络的守护者。它不断监视交易流，并执行交易以计算状态转换。如果计算的状态结果与提议的状态不匹配，则挑战者将请求欺诈证明并将其提交仲裁。 （有关更多信息，请参见Fraud Proof。）
- **ZK Prover** - ZK Prover根据批处理的交易生成状态转换的ZK证明。证明将发送到仲裁层以优化安全机制所需的挑战期。(有关更多信息，请参见使用[Hybrid Security with ZK Proof](/docs/tech-highlights/hybrid-security))
- **提议者** - 提议者负责执行所有排序后的交易并将生成的状态提交给仲裁层。

## DA层

DA（数据可用性）的重要性在于确保任何人都可以访问发生在离线环境中的所有交易。

Rooch网络将采用第三方解决方案以确保交易的可用性并保持高TPS。我们仍在研究不同的选择，这其中包括[Celestia](https://celestia.org/)，[Eigenlayer](https://www.eigenlayer.xyz/)，[Polygon Avail](https://polygon.technology/blog-tags/polygon-avail)，以及[Arweave](https://www.arweave.org/).

## 仲裁层

仲裁层作为“法院”，为仲裁和验证包含不同DApp的不同Rooch实例中发生的所有状态转换。

在Rooch网络中，以太坊承担了仲裁层的角色，并将是Rooch生态系统中所有DApp的唯一安全来源。这里的理由是以太坊具有市值测量的最高实际攻击成本。

在以太坊上，有三个智能合约担任不同的角色：

- **欺诈证明仲裁合约** - 该合约接受欺诈证明，并模拟状态转换的执行以做出削减决策。
- **ZK Verifier合约** - 该合约负责验证由ZK Prover提交的ZK证明。
- **State Commitment Chain** - 该合约保留由Proposer提交的所有状态的记录链。

## 结算层

结算层为所有基于Rooch的DApp提供状态结算。使用Rooch作为执行环境的每个DApp都能够在其DApp和不同的Layer1之间迁移资产（即token）。

Rooch网络支持将任意Layer1整合到结算层中，所有使用以太坊作为安全保证源的内容。(有关更多信息，请参见[Multi-Chain Settlement](/docs/tech-highlights/multi-chain-settlement))
